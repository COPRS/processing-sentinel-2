# Resubmit a processing for S2 L0c

In general, to resubmit an error, you need to go back from that error to the previous(es) processing steps.

If the need is to resubmit an L0C error, you will *replay* the processing message of the associated L0U.

## Search the L0C NOK processing in Kibana

Connect to Kibana and search the NOK processing on the processing trace index with the following filter :
![Kibana filter l0c](/docs/media/resubmit1.png)

Adjust your filter (time windows or task id) to isolate the NOK processing you want to resubmit. It should return only one hit :
![Kibana hits](/docs/media/resubmit2.png)

Save the L0U datastrip's name. In this example :
`S2A_OPER_MSI_L0U_DS_REFS_20230505T093732_S20230504T095149_N00.00`

## Search the session ID for the L0U in Kibana

Edit the filter to search with the `ew-l0u` application instead of `ew-l0c`, status `OK` instead of `NOK` and with the l0u datastrip as `task.output.filenames.filename`:
![Kibana filter l0u](/docs/media/resubmit3.png)

Extend the result and retrieve the session ID from the `task.input.filenames.filename` :
![Kibana session](/docs/media/resubmit4.png)

The session ID in this example is `S2A_20230504111308041075`.

## Search the processing message for the L0U session in Kafka

Search for a message with the session ID on the output topic of the L0U preparation worker :

```Console
kubectl -n infra exec -ti kafka-cluster-kafka-0 -c kafka -- \
  bash /opt/kafka/bin/kafka-console-consumer.sh \
  --bootstrap-server kafka-cluster-kafka-bootstrap.infra.svc.cluster.local:9092 \
  --topic s2-l0u-part1.pw-l0u \
  --from-beginning \
  | grep S2A_20230504111308041075
```
